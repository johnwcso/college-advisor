Project Structure
All files live inside college-advisor/. Move this directory anywhere — just update the cron path and config/settings.json output_dir.
college-advisor/
System Layers
Each layer is a discrete, independently runnable module. Run them in sequence or individually.
Layer 1a — Intake
Student Profile Builder
Interactive CLI interview (or questionnaire form) that collects academic, geographic, financial, activity, and strategy preferences from a new student.
Inquestionnaire.json + user input
Outdata/student_profiles/profile_<id>.json
intake/intake_cli.pyintake/questionnaire.json
Layer 1b — Discovery
College List Generator
Scores and filters the base college catalog against the student profile. Proposes reach/match/safety tiers and generates the seed URLs for the crawler config.
Inprofile_<id>.json + catalog.json
Outupdates config/colleges.json
discovery/discover_colleges.pyconfig/catalog.json
Layer 2 — Crawl
Playwright Web Crawler
Visits official college admissions, academics, aid, and athletics pages using a real browser. Handles JS-heavy pages, follows links within allowed domains, and stores rendered text and page metadata.
Inconfig/colleges.json
Outdata/crawl_output/<college>/
crawler/college_crawl.pyconfig/settings.json
Layer 3 — Verify
Data Verification
Checks that crawl output is structurally correct, extracts key signals (deadlines, tuition, test policy), and detects content drift between crawl runs. Flags anomalies for review.
Indata/crawl_output/
Outverification report + console flags
verification/verify_college_data.py
Layer 4 — Improve
Auto-Improve Engine
Analyzes crawl history and verification signals to propose structured actions: add seeds, tighten regexes, adjust crawl frequency, or flag colleges that no longer fit a student's profile.
Indata/crawl_output/ + profiles
Outoutput/actions/actions_<date>.json
improve/auto_improve.py
Layer 5 — View
HTML Viewer
Generates a self-contained, searchable HTML dashboard from all crawl output. No server required — open the file directly in any browser.
Indata/crawl_output/
Outoutput/viewer.html
viewer/generate_viewer.py
Schema Files
All data files are validated against these JSON Schemas. Located in schema/.
student_profile.schema.json
Validates data/student_profiles/profile_*.json. Generated by intake_cli.py.
student_idstringUnique ID e.g. uuid
academicobjectcurriculum, scores, majors, selectivity
geographyobjectcountries, regions, campus setting
financialobjectbudget_band, aid_required, loans
activitiesobjecthooks, priority, recruiting interest
experience_preferencesobjectsize, teaching style
constraintsobjecthard_filters, special_notes
application_strategyobjecttarget_list_size, reach/match/safety counts
college_catalog.schema.json
Validates each entry in config/catalog.json. Used for discovery scoring.
idstringSlug e.g. harvard-university
namestringDisplay name
country / regionstringISO country + region label
selectivityenumhighly selective / selective / …
majors_availablestring[]List of offered disciplines
sailing_teamobjecthas_team, level (varsity/club), conference
aidobjectneed_blind, meets_full_need, merit
official_seedsuri[]Admissions/program URLs for crawler
crawl_page.schema.json
Validates each JSONL line in data/crawl_output/*/pages.jsonl.
collegestringCollege name
url / final_urluriRequested and resolved URL
statusintegerHTTP status code
depthintegerCrawl depth from seed
titlestringPage <title>
textstringRendered body text (post-JS)
linksuri[]All links extracted from page
crawled_atdate-timeISO 8601 timestamp
errorstring?Null if success
action.schema.json
Validates output/actions/actions_*.json emitted by auto_improve.py.
run_idstringe.g. 2026-04-22-nightly
generated_atdate-timeISO 8601 timestamp
actions[].typeenumcrawl_seed_add, regex_update, recrawl_frequency_change, student_list_flag, …
actions[].priorityenumcritical / high / medium / low
actions[].reasonstringHuman-readable explanation
actions[].proposed_valueanyNew value to apply (regex, URL, days, etc.)
actions[].auto_applybooleanSafe to apply without review
End-to-End Flow
Run each step in sequence for a new student, or run steps individually for existing profiles.
1a. Student Intake
Interview → profile_<id>.json
intake/intake_cli.py
↓
1b. College Discovery
Score catalog → update colleges.json
discovery/discover_colleges.py
↓
2. Crawl
Playwright → crawl_output/
crawler/college_crawl.py
↓
3. Verify
Structural + extraction + drift checks
verification/verify_college_data.py
↓
4. Auto-Improve
Emit action proposals → actions_<date>.json
improve/auto_improve.py
↓
5. View
Build viewer.html from crawl output
viewer/generate_viewer.py
↓
🔁 Weekly Cron Refresh
college_crawl.py --interval 7 → re-runs crawl + verify + improve
crontab
